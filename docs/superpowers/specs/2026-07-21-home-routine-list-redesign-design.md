### 진행률 링 (공통 위젯)

- 40×40, stroke 4, 트랙 `#C9D6D4`, 진행 `#55CFBA`, 중앙 `NN%` (12/w800, `#55CFBA`)
- 100% → 링 대신 채워진 체크 원(`#55CFBA` + 흰 체크)
- 진행률 = 완료 step 수 / 전체 step 수. 완료 여부는 서버 `completed` ∪ 로컬 `childRoutineProvider` 체크

## 서버에 요청할 것 — 이슈 #71

클라 작업의 선행 조건이 **아니다.** 현재 서버 계약만으로 #69는 성립한다.

1. `RoutineStepResponse`에 `title` — Figma가 제목과 설명을 나눠 보여준다
   (현재는 `description` 하나뿐이라 카드 제목과 본문이 같은 문장이 된다 — #70과 연관)
2. `RoutineResponse`에 `totalStepCount` · `completedStepCount` · `progressRate`(0~100 정수)
   — 반올림 규칙을 서버에 고정한다. **우선순위 낮음** (클라 계산으로 이미 동작)
3. **`cancel`의 별 회수(−1) 재검토** — `RoutineService.java:215`가 별을 회수한다.
   아이는 아무 카드나 체크를 풀 수 있어, 실수 한 번에 모은 별이 줄어든다.
   `totalStars` 음수 하한 가드도 없다

> 별 API 신설은 요청하지 않는다. `Member.totalStars` · `MemberResponse.totalStars` ·
> `complete` +1 / `cancel` −1 로직이 **이미 서버에 있다.**
