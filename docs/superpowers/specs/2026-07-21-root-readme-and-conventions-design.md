# 루트 README 추가 및 프로젝트 공통 규칙 정립

> 작성일: 2026-07-21
> 상태: 승인됨

## 배경

main 브랜치 push 시 `README VERSION UPDATE` 워크플로우가 계속 실패한다.
([run 29802868878](https://github.com/Twin-Fang/elum/actions/runs/29802868878))

동시에 이슈 제목 규칙이 팀 내에서 정립되지 않아 `[작업]`, `[admin]`, `[client]` 등
템플릿에 없는 표기가 섞여 있었다.

## 문제 1 — 워크플로우 실패

### 원인

`.github/workflows/PROJECT-COMMON-README-VERSION-UPDATE.yaml`은 `README.md 파일 버전 정보
업데이트` 스텝에서 루트 `README.md`를 대상으로 `grep`·`sed -i`를 실행한다.

이 저장소에는 **루트 README.md가 없다.** `docs/README.md`는 존재하지만 워크플로우는
루트만 참조하므로 해당 스텝이 실패한다.

### 해결

루트에 `README.md`를 추가한다. **워크플로우는 수정하지 않는다.**

워크플로우에 "파일이 없으면 스킵" 같은 방어 로직을 넣지 않는 이유: README가 사라진 상황을
조용히 통과시키면 버전 표기가 갱신되지 않는 것을 아무도 알아채지 못한다. 실패로 드러나는
편이 낫다.

### 버전 섹션 처리

워크플로우는 버전 라인이 없으면 자동으로 삽입한다(`sed -i '/^# /a\...'`). 하지만 README를
만들 때 버전 섹션을 **미리 포함**한다.

- 형식이 로컬에서 눈에 보인다
- 첫 실행에서 sed 삽입 위치가 어긋날 여지를 없앤다

형식은 워크플로우가 인식하는 패턴을 따른다:

```markdown
<!-- AUTO-VERSION-SECTION: DO NOT EDIT MANUALLY -->
## 최신 버전 : v1.0.23 (2026-07-21)
```

제약(워크플로우 주석 기준):
- 마크다운 볼드(`**`) 사용 금지
- 정규표현식 특수문자(`*`, `[`, `]`, `^`, `$`) 사용 금지
- 콜론(`:`) 뒤에 버전이 바로 와야 함

`docs/`에 이미 상세 기획 문서(01~07)가 있으므로 README는 **재작성이 아니라 요약**이며,
깊은 내용은 링크로 넘긴다. 중복 서술을 만들지 않는다.

## 문제 3 — 이슈 제목 규칙 부재

### 규칙

이 저장소는 모노레포이므로 카테고리를 **2단계**로 나눈다.

```
{이모지}[{유형}][{모듈}][{주제}] {설명}
```

| 구분 | 값 |
| --- | --- |
| 유형 | `버그` `기능요청` `기능추가` `기능개선` `디자인` `시험요청` (템플릿 기준) |
| 모듈 | `클라` `서버` `관리자` |
| 주제 | `홈` `온보딩` `카드` `설정` 등 짧은 명사 |

원칙:
- 카테고리는 **한글**로 쓴다 (`client` → `클라`, `admin` → `관리자`)
- 주제에 **"화면"을 붙이지 않는다** (`홈화면` → `홈`)
- 모듈은 축약형을 쓴다 (`클라이언트` → `클라`)

예시:
```
⚙️[기능추가][클라][홈] ...
⚙️[기능추가][서버][관리자] ...
```

### 기존 이슈 정정

| # | 변경 후 |
| --- | --- |
| 1 | `⚙️[기능추가][클라][온보딩] 초기 아키텍처 구축 및 온보딩 구현` |
| 2 | `⚙️[기능추가][서버][관리자] daisyUI 개편 및 조회 기능 추가` |
| 3 | `⚙️[기능추가][클라][설정] 환경설정 분리(.env) 및 네트워크 계층 구축` |

#2는 관리자 페이지가 Spring 서버 렌더링(daisyUI)이므로 모듈을 `서버`로 판단했다.

## 문제 4 — 루트 CLAUDE.md 부재

`client/CLAUDE.md`, `server/CLAUDE.md`는 있으나 루트에는 없어 저장소 전역 규칙을 둘 곳이 없다.

루트 `CLAUDE.md`를 신규 생성하고 다음을 기록한다:

1. 이슈 제목 규칙 (위 2단계 카테고리)
2. 루트 README의 버전 라인은 워크플로우가 자동 관리 — **수동 편집 금지**
3. 모노레포 구조 안내 (client / server / docs)

하위 `CLAUDE.md`와 역할이 겹치지 않게, 루트에는 **저장소 전역 규칙만** 둔다.
기술 스택·코딩 규칙은 각 모듈 CLAUDE.md에 그대로 남긴다.

## 작업 순서

1. 루트 `README.md` 생성 (버전 섹션 포함)
2. 루트 `CLAUDE.md` 생성
3. 이슈 #1·#2·#3 제목 정정
4. 커밋 → main push → 워크플로우 통과 확인

## 검증 기준

- `README VERSION UPDATE` 워크플로우가 성공한다
- 워크플로우가 `## 최신 버전 :` 라인을 정상적으로 갱신한다 (실패도, 중복 삽입도 없이)
