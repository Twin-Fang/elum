## API를 잠시 끊는다 ⚠️

`POST /api/routines`가 **3회 중 1회 502**로 불안정하다(이슈 #34). 카드가 나와야
이후 화면을 만들 수 있는데 매번 실패하면 개발이 막힌다.

**`AppConfig.useMock`을 켜서 로컬 카드로 진행한다.** 이미 있는 스위치이므로
코드를 주석 처리하지 않는다 — 주석은 되돌리는 것을 잊는다.

```dart
// .env
ELUM_USE_MOCK=true   // 서버 안정화되면 false로 되돌린다 (이슈 #34)
```

호출 코드는 그대로 두고 실행 경로만 바꾸는 것이므로, 서버가 고쳐지면
`.env` 한 줄로 복구된다.

다만 **mock 카드가 Figma와 달라** 화면이 빈약해 보인다. `DemoCards`를 Figma
`보호자_홈_최근일과`(309:3739)의 5장으로 맞춘다.

| 순서 | 제목 | 설명 |
| --- | --- | --- |
| 1 | 옷을 입어요 | 학교에 갈 옷을 차례대로 입어요 |
| 2 | 여벌 양말을 챙겨요 | 가방에 여벌 양말을 넣어요 |
| 3 | 우비를 입어요 | 비에 젖지 않도록 우비를 입어요 |
| 4 | 우산을 챙겨요 | 현관에서 우산을 챙겨요 |
| 5 | 천천히 학교로 가요 | 비 오는 길에서는 천천히 걸어요 |

**카드에 제목과 설명이 따로 있다.** 현재 `ActionCard`는 `description` 하나뿐이라
필드를 추가해야 한다. 서버 `RoutineStepResponse`에도 제목이 없으므로, 서버가
줄 때까지는 설명의 첫 구절을 제목으로 쓰지 않고 **로컬 필드로만** 둔다.

## 화면별 설계

### 1. 카드확인 (262:5124)

가로로 넘기는 카드 스택이다. Figma가 카드 3장을 겹쳐 배치했고 뒤쪽 카드가
살짝 보인다(345×431, 뒤 카드는 x가 밀려 있음).

```
✨ 카드 5개가 생성되었어요
   내용을 확인하고 카드를 수정하거나 삭제해주세요

   ┌──────────────┐ ← 카드 스택 (PageView)
   │  [이미지]  수정│
   │  1 옷을 입어요  │
   │  🔊 학교에 갈... │
   └──────────────┘
        [저장하기]
```

**카드 색이 순서마다 다르다.** Figma에서 1번 민트(`#B5EAEC`/`#93DBCC`),
2번 파랑(`#CED8FF`/`#9CADF1`), 3번 복숭아(`#FFDAC7`/`#EB9B73`). 홈의 추천 타일과
같은 방식으로 인덱스에 따라 배정한다.

**이미지는 표시할 수 없다.** 서버 `imagePath`가 404다(이슈 #20). 자리를 비워두면
카드가 무너지므로 캐릭터 일러스트로 대체한다. 서버가 고쳐지면 교체한다.

`수정` 버튼은 이번 범위에서 **자리만 만든다.** 편집 화면이 Figma에 없다.

### 2. 보호자_홈_최근일과 (309:3739)

기존 보호자 홈에 두 가지가 바뀐다.

- 섹션 제목 `최근 일과` → **`오늘 일과`**
- 빈 상태 대신 **카드 5장이 리스트로** (각 68px, 번호 배지 + 제목 + 설명 + 완료 체크)

번호 배지 색이 5종으로 다르다(`#93DBCC`, `#9CADF1`, `#EB9B73`, `#F3C500`, `#C7EB73`).

우측 상단 캐릭터 배지를 누르면 **PIN 화면으로** 간다.

### 3. PIN 전환 (309:2837)

**양방향으로 쓰인다.** Figma에 두 벌이 있고 문구만 다르다.

| 방향 | 문구 |
| --- | --- |
| 보호자 → 아이 | 암호를 입력하면 아이 화면으로 전환돼요 |
| 아이 → 보호자 | 암호를 입력하면 보호자 화면으로 전환돼요 |

한 화면에 **목적지를 파라미터로** 받아 문구를 바꾼다. 두 파일로 나누면
PIN 입력 로직이 복제된다.

온보딩에서 저장한 PIN과 비교하고, 틀리면 점만 비운다. **아동도 보는 화면이라
빨강·경고 아이콘을 쓰지 않는다.**

### 4. 아이_홈 (309:3548 / 309:3648)

두 프레임의 차이는 **체크 상태 하나**다. 별도 화면이 아니라 같은 화면의 상태다.

```
[로고]              [캐릭터 배지]
   ┌──────────────┐
   │  [이미지]      │  ← 카드 스택 (가로 스와이프)
   │  1 옷을 입어요  │
   │  🔊 학교에 갈... │
   └──────────────┘
        (  ✓  )      ← 88×88 체크 버튼
```

**터치 타겟이 88×88이다.** 아동 모드 최소 64×64 규칙을 넉넉히 만족한다.

체크 전에는 테두리만 있는 원(`#C9D6D4` 8px), 체크 후에는 채워진다.

### 5. 보상 (309:4055 / 334:4320 / 343:4434)

어두운 그라데이션 배경(`#0C0D1A` → `#242634`)에 빛나는 별과 캐릭터.

**세 화면을 랜덤으로 보여준다.** 캐릭터 3종(루미·포포·루루)의 문구가 다르다.

| 캐릭터 | 제목 | 문구 |
| --- | --- | --- |
| 루미 | 축하해요! | 할 일을 해내서\n루미가 별을 가져왔어요 |
| 포포 | 잘했어요! | 할 일을 해내서\n포포가 별을 가져왔네요 |
| 루루 | 잘했어요! | 할 일을 해낸 |

> 루루 문구가 `할 일을 해낸`으로 끊겨 있다. Figma 미완성으로 보이므로
> 포포와 같은 형식(`할 일을 해내서\n루루가 별을 가져왔네요`)으로 맞춘다.
> 디자이너 확인 후 교체한다.

**언제 뜨는가** — 카드를 체크할 때마다 뜬다. 단 **같은 카드를 다시 체크하면
뜨지 않는다.** 체크 해제 후 재체크에 또 축하하면 보상이 가벼워진다.

이를 위해 카드별로 "보상을 이미 받았는가"를 기억한다. 완료 여부(`completed`)와는
다른 값이다 — 해제해도 보상 이력은 남는다.

## 애니메이션

`docs/motion.md`를 따른다. 보상 화면은 **브랜드 핵심 순간**이라 강한 인터랙션을
허용하는 유일한 자리다(토스 5원칙 中 "강한 인터랙션은 선택적").

| 요소 | 동작 |
| --- | --- |
| 별 | scale 0 → 1.15 → 1.0 (elasticOut, 600ms) |
| 작은 별 3개 | 순차 등장 (stagger 100ms) |
| 문구 | fade + slide up (`AppMotion.normal`) |
| 카드 체크 | 원이 채워지며 scale 반동 |

**접근성 설정을 켠 사용자는 애니메이션 없이 결과만** 보여준다.
아동 모드에서 특히 중요하다.

## 데이터

카드 완료 상태는 **로컬에만** 둔다. 서버 `PATCH /steps/{id}/complete`가 있지만
`POST /api/routines`가 불안정한 상황이라 지금 붙이면 검증이 불가능하다.
`useMock`을 끄는 시점에 함께 붙인다.

## 실패 경로

| 상황 | 처리 |
| --- | --- |
| 저장할 카드가 없음 | 카드확인 화면에 진입하지 않고 홈으로 |
| PIN 미설정 | 검증 없이 통과 (온보딩을 건너뛴 개발 상태) |
| PIN 불일치 | 점만 비우고 머문다. 경고색 금지 |
| 아이 홈에 일과 없음 | 빈 상태 안내 |
| 보상 이미지 로드 실패 | 별만 보여준다 |

## 테스트

- 카드확인: 카드 수 표시, 스와이프, 색 배정, 저장 → 홈 이동
- 홈: `오늘 일과` 문구, 카드 5장 리스트, 배지 색 5종
- PIN: 방향별 문구, 일치 시 이동, 불일치 시 경고색 없음
- 아이 홈: 88×88 터치 타겟, 체크 → 보상, 재체크 시 보상 없음
- 보상: 3종 랜덤, 캐릭터별 문구, 접근성 시 애니메이션 없음
