# 카드 읽어주기 (TTS)

- 작성일: 2026-07-21
- 관련 화면: 카드확인(`262:5124`) · 아이_홈(`309:3548`)

## 배경

행동 카드에 스피커 아이콘이 있지만 **누르면 아무 일도 일어나지 않는다.**
`ActionCardView.onSpeak`가 있으나 어느 화면도 넘기지 않아 죽은 버튼이다.

읽어주기는 장식이 아니다. 이룸의 카드는 `이미지·음성 안내·체크리스트` 세 축으로
설계됐고(docs/README.md), **글을 못 읽는 아동에게는 음성이 유일한 정보 경로**다.

## 실 서버 검증 결과

`ai.suhsaechan.kr`에 직접 요청해 확인했다.

```bash
GET /api/flask/tts/engines
→ supertonic  running  gpu=false     ← CPU 전용이라 항상 켜져 있다
   cosyvoice   stopped  gpu=true
   qwen3tts    stopped  gpu=true

POST /api/flask/tts  {"text":"옷을 입어요. 학교에 갈 옷을 차례대로 입어요",
                      "engine":"supertonic","voice":"F1"}
→ 200  audio/wav  374,828 bytes  (16bit mono 44.1kHz)
```

**Supertonic을 쓴다.** GPU 엔진은 "한 번에 1개" 정책이라 대부분 꺼져 있는데,
이것만 CPU 전용이라 그 정책의 예외다(`tts_config.py:41 gpu: False`).

| 항목 | 값 |
| --- | --- |
| 인증 | `X-API-Key` 헤더 |
| 텍스트 제한 | 500자 |
| 응답 | `audio/wav` |

## 두 갈래 — 기기 우선, 서버 폴백

```
읽기 버튼
  ├─ 1순위: flutter_tts (기기 내장)
  └─ 2순위: Supertonic 서버 (WAV 다운로드 → 재생)
```

**기기 TTS를 먼저 쓰는 이유** — 네트워크 없이 동작하고, 375KB를 매번 받지 않아도
된다. 지연도 훨씬 짧다.

**서버가 필요한 이유** — 에뮬레이터나 한국어 음성이 없는 기기에서는 기기 TTS가
조용히 실패한다. 그때 소리가 아예 안 나면 기능이 없는 것과 같다.

## 구조

```
SpeechService (인터페이스)
  ├─ DeviceSpeech    flutter_tts
  ├─ RemoteSpeech    Supertonic + audioplayers
  └─ FallbackSpeech  둘을 순서대로 시도
```

화면은 `SpeechService` 하나만 안다. **어느 경로로 소리가 났는지 알 필요가 없다.**
나중에 엔진을 바꿔도 화면 코드는 그대로다.

### 읽는 내용

제목과 설명을 마침표로 잇는다.

```
"옷을 입어요. 학교에 갈 옷을 차례대로 입어요"
```

제목만 읽으면 무엇을 어떻게 해야 하는지가 빠지고, 설명만 읽으면 화면에 크게
보이는 제목과 어긋나 혼란스럽다.

서버 제한이 500자인데 카드 문장은 훨씬 짧다. 그래도 넘치면 잘라 보낸다 —
길이 때문에 실패하는 것보다 앞부분이라도 들리는 편이 낫다.

### 재생 중 다시 누르면 멈춘다

아동이 여러 번 누를 때 소리가 겹치면 알아들을 수 없다. 같은 버튼이 재생/정지를
토글하고, 재생 중에는 아이콘 상태가 바뀐다.

**화면을 벗어나면 멈춘다.** 소리가 남아 다음 화면까지 따라오면 안 된다.

## 실패 처리

| 상황 | 처리 |
| --- | --- |
| 기기에 한국어 음성 없음 | 서버로 폴백 |
| 서버 엔진 정지(503) | 안내 문구 |
| 네트워크 없음 | 기기 TTS만 시도 |
| 둘 다 실패 | `소리를 재생할 수 없어요 (E-TTS)` |
| 화면 이탈 | 재생 중지 |

**버튼은 항상 보인다.** 숨기면 기기마다 UI가 달라져 디자인이 흔들리고, 일시적
실패인지 영구적 불가인지 구분할 방법도 없다.

에러 코드를 노출한다. 아동은 못 읽지만 **보호자가 제보할 때 필요하다.**

## 아이콘 버그도 함께 고친다

카드의 스피커 아이콘이 찌그러져 보인다. SVG가 25×25인데 Figma 배치는 24×24라,
크기만 지정하면 비율이 눌린다. `BoxFit.contain`으로 비율을 지킨다.

## 보안 — API 키

**코드에 박지 않는다.** `.env` → `AppConfig` 경유로 기존 규칙을 따른다.

| 파일 | 처리 |
| --- | --- |
| `.env` | 실제 키 (커밋 안 함) |
| `.env.example` | 빈 값 + 설명 |
| GitHub Secret `CLIENT_ENV_FILE` | 배포용 |

키가 없으면 서버 폴백만 비활성되고 **기기 TTS는 그대로 동작한다.** 설정 하나
때문에 기능 전체가 죽지 않는다.

## 테스트

- 제목·설명을 이어 붙인다
- 500자를 넘기면 자른다
- 기기 TTS가 실패하면 서버로 넘어간다
- 둘 다 실패해도 예외를 던지지 않는다
- 재생 중 다시 부르면 멈춘다
- API 키가 없으면 서버를 호출하지 않는다
- 화면을 벗어나면 정지한다

## 범위 밖

- 음성 종류·속도 설정 UI (`architecture.md` 미확정 사항)
- 자동 재생 — 카드를 넘길 때마다 소리가 나면 산만하다
- 보이스 클로닝 — 서버는 지원하지만 이 앱에 쓸 자리가 없다
