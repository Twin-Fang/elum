# 03. 화면 플로우 및 페이지별 명세

> 상태: 초안 (브레인스토밍 정리본) — 플로우와 화면 목적은 합의됨, 문구·레이아웃은 러프.
> 회원가입·로그인은 MVP에서 구현하지 않는다.

## 전체 플로우

```
시작 화면
  ↓
아이 호칭 설정          (온보딩 1/2)
  ↓
도움 목표 선택          (온보딩 2/2)
  ↓
AI 맞춤 설정 완료       ← 첫 번째 와우 포인트
  ↓
보호자 홈
  ↓
새 일과 입력
  ↓
AI 분석 (DLP 보호 단계 포함 로딩)   ← 두 번째 와우 포인트 (보안)
  ↓
AI 추가 질문            ← 세 번째 와우 포인트 (목표 반영 질문)
  ↓
행동 카드 생성 → 보호자 검토·승인
  ↓
아이 화면으로 전환 (바텀시트)
  ↓
행동 카드 수행 (카드 데이터만 바꿔 반복)
  ↓
별 보상 완료
```

필수 화면은 총 **10종 내외**. 아동 수행 화면은 같은 화면을 카드 데이터만 바꿔 반복한다.

## 화면 요약표

| # | 화면 | 핵심 목적 | 주요 버튼 | AI 사용 | MVP |
| --- | --- | --- | --- | --- | --- |
| 1 | 시작 화면 | 서비스 가치 전달 (5초 안에 이해) | 이룸 시작하기 | 없음 | 필수 |
| 2 | 아이 호칭 설정 | 최소 개인화 | 다음 | 없음 | 필수 |
| 3 | 도움 목표 선택 | AI 카드 생성 기준 설정 | 맞춤 설정하기 | 목표값 저장 | 필수 |
| 4 | AI 맞춤 설정 완료 | 개인화 체감 | 첫 일과 만들기 | 규칙 매핑(실호출 없음) | 강력 추천 |
| 5 | 보호자 홈 | 일과 생성 진입점 | 새로운 일과 만들기 | 없음 | 필수 |
| 6 | 새 일과 입력 | 자연어 상황 입력 | AI로 행동 카드 만들기 | 입력 분석 | 필수 |
| 7 | AI 분석 로딩 | DLP 보호 + 생성 과정 표현 | 없음 | 실제·연출 병행 | 필수 |
| 8 | AI 추가 질문 | 부족한 생활 맥락 확인 | 이 내용으로 카드 만들기 | 목표 기반 질문 생성 | 필수 |
| 9 | 카드 검토·승인 | 보호자 통제(수정·승인) | 아이 화면으로 시작하기 | 출력 형식 검증 | 필수 |
| 10 | 아이 화면 전환 | 모드 전환 안내 (바텀시트/팝업) | 시작 | 없음 | 필수 |
| 11 | 아동 카드 수행 | 이미지+TTS+체크리스트 수행 | 했어요! | 보호된 결과만 사용 | 필수 |
| 12 | 별 보상 완료 | 성취 경험 | 완료하기 / 카드 다시 보기 | 없음 | 필수 |

## 페이지별 러프 명세

### 1. 시작 화면

| 구분 | 내용 |
| --- | --- |
| 화면 목적 | 이룸이 어떤 앱인지 5초 안에 이해시키기 |
| 상단 | 이룸 로고 또는 심볼 |
| 메인 문구 | **아이의 하루를 이해하기 쉬운 행동 카드로** |
| 설명 문구 | 보호자가 입력한 일과를 이미지와 음성 안내가 포함된 행동 카드로 만들어드려요. |
| 이미지 | 보호자와 아동, 행동 카드 3장이 연결되는 일러스트 |
| 하단 버튼 | **이룸 시작하기** |

### 2. 아이 호칭 설정 (온보딩 1/2)

| 구분 | 내용 |
| --- | --- |
| 질문 | **아이를 어떻게 불러드릴까요?** |
| 보조 문구 | 행동 카드와 음성 안내에서 사용할 이름이에요. |
| 개인정보 안내 | 🔒 실제 이름 대신 평소 사용하는 별명을 입력해도 괜찮아요. |
| 입력 예시 | 하늘이 |
| 버튼 활성화 조건 | 한 글자 이상 입력 |
| 저장값 | `nickname: "하늘이"` |

```
←                         1 / 2

아이를 어떻게
불러드릴까요?

행동 카드와 음성 안내에서
사용할 이름이에요.

[ 하늘이                         ]

🔒 실제 이름 대신 평소 사용하는
   별명을 입력해도 괜찮아요.

[ 다음 ]
```

### 3. 도움 목표 선택 (온보딩 2/2)

| 구분 | 내용 |
| --- | --- |
| 화면 목적 | 보호자가 원하는 도움 방식을 **AI 카드 생성 기준**으로 설정 |
| 질문 | **하늘이의 어떤 순간을 도와주고 싶나요?** |
| 보조 문구 | 여러 개를 선택할 수 있어요. (복수 선택) |
| 버튼 | **맞춤 설정하기** |
| 저장값 | `supportGoals: [...]` — enum은 [04-ai-card-generation.md](./04-ai-card-generation.md) 참고 |

선택 항목 (4종):

- 🧩 해야 할 일을 순서대로 이해해요
- 🎒 필요한 준비물을 스스로 챙겨요
- 🌱 새로운 상황을 미리 준비해요
- ⭐ 혼자 끝까지 해내는 경험을 만들어요

데모 선택값: 준비물 챙기기 + 새로운 상황 준비 (2개)

**중요**: 이 선택은 장식이 아니라 실제 카드 생성에 반영되어야 한다.

| 선택한 목표 | 행동 카드 반영 |
| --- | --- |
| 순서 이해 | 한 카드에 행동 하나, 순서 강조 |
| 준비물 챙기기 | 준비물 질문과 체크리스트 생성 |
| 새로운 상황 준비 | 평소와 달라지는 점·주의사항 추가 |
| 스스로 수행하기 | 보호자 지시보다 아동 중심 문장 사용 |

### 4. AI 맞춤 설정 완료 (첫 번째 와우 포인트)

목표 선택 직후 바로 홈으로 보내지 않고, **AI가 맞춤 기준을 준비했다는 화면**을 한 장 보여준다.

- 선택한 목표에 따라 안내 문구를 매핑해서 표시한다. 예:
  - `PREPARE_ITEMS` 선택 → "준비물은 체크리스트로 보여드려요."
  - `PREPARE_NEW_SITUATIONS` 선택 → "평소와 달라지는 상황은 미리 알려드려요."
- 공통 안내:
  - 한 카드에는 하나의 행동만 담아요.
  - 👪 보호자가 확인한 뒤 아이에게 전달해요.
  - 🔒 장애 유형이나 진단명 없이 선택한 도움 방식만 활용합니다.
- 하단 버튼: **하늘이의 첫 일과 만들기**
- **이 화면에서는 AI API를 호출하지 않는다.** 선택값 → 문구 매핑만으로 충분하다. (사용자에게는 AI 맞춤 준비 과정으로 보이지만 구현은 단순·안전)
- 주의: "장애 특성을 분석했어요", "행동 패턴을 파악했어요" 같은 **과장 문구는 쓰지 않는다** (실제로 하지 않는 일).

### 5. 보호자 홈

| 구분 | 내용 |
| --- | --- |
| 상단 인사 | 안녕하세요, 하늘이 보호자님 |
| 메인 질문 | 오늘은 어떤 일과를 준비할까요? |
| 핵심 CTA | **새로운 일과 만들기** |
| 보조 요소 | 추천 일과 카드(`RoutineSuggestionCard`), 기존 일과 목록 (러프) |

데모 팁: 추천 카드 자동 입력을 쓰는 것이 안전하고, 문장 입력 기능을 보여주기 위해 마지막 단어 정도만 직접 수정한다.

### 6. 새 일과 입력

| 구분 | 내용 |
| --- | --- |
| 질문 | **하늘이에게 어떤 준비가 필요한가요?** |
| 안내 | 평소 이야기하듯 입력해주세요. AI가 하늘이가 수행할 수 있는 작은 행동으로 나누어드려요. |
| 입력창 | 여러 줄 자연어 입력 |
| 입력 예시 | 내일 비가 많이 올 예정이야. 아이가 학교에 갈 수 있게 준비해야 해. |
| 추천 태그 | 등교, 외출, 병원, 새로운 장소 |
| 하단 버튼 | **AI로 행동 카드 만들기** |

하단에 작은 보안 카드 배치:

```
🛡 AI DLP로 안전하게 보호해요

입력한 내용은 개인정보 보호 과정을 거친 뒤,
행동 카드 생성에 필요한 정보만 AI에 전달돼요.

[ 어떻게 보호하나요? ]
```

### 7. AI 분석 로딩 (DLP 보호 단계 포함 — 두 번째 와우 포인트)

DLP 보호를 별도 페이지로 빼는 안도 있었으나, **로딩 화면의 3단계 진행 연출로 통합**하는 방향을 추천. (별도 페이지 안은 [05-ai-dlp-gateway.md](./05-ai-dlp-gateway.md) 참고)

```
하늘이의 행동 카드를 안전하게 준비하고 있어요

✓ 입력한 상황을 확인했어요
✓ AI DLP로 개인정보를 보호했어요
  이름과 학교 정보는 안전하게 가렸어요.     [보호 내역]
● 하늘이에게 필요한 질문을 준비하고 있어요

🔒 원문은 AI에 그대로 전달되지 않아요.
```

- 연출 타이밍(러프): 0초 상황 확인 → 1.5초 DLP 보호 완료 → 1.6초 질문 준비 → 2.5초 다음 화면 자동 이동
- 실제 AI 응답이 빨라도 **최소 1~2초는 보여준다** (보안 처리가 체감되도록). 오래 걸리면 마지막 단계에서 계속 로딩.
- `보호 내역` 링크 → 바텀시트로 전송 전/후 문장 비교 표시 (일반 사용자는 안 눌러도 됨, 심사 데모용). 상세는 [05-ai-dlp-gateway.md](./05-ai-dlp-gateway.md).

### 8. AI 추가 질문 (세 번째 와우 포인트)

온보딩에서 선택한 목표가 **실제 질문으로 연결**되는 장면.

| 구분 | 내용 |
| --- | --- |
| 상단 배지 | AI가 추가 정보를 확인하고 있어요 |
| 연결 안내 | 하늘이가 선택한 목표에 맞춰 준비물과 평소 달라지는 점을 먼저 확인하고 있어요. |
| 질문 | **하늘이가 비 오는 날 평소와 다르게 챙겨야 하는 물건이 있나요?** |
| 선택 방식 | 복수 선택 칩 또는 카드 (`SelectableChip`) + 직접 입력 |
| 선택지 예시 | 우산, 우비, 장화, 여벌 양말, 작은 수건, 직접 입력 |
| 데모 선택값 | 우산, 우비, 여벌 양말 |
| 하단 버튼 | **이 내용으로 카드 만들기** |

주의: "필요한 준비물이 있나요?" 같은 **일반적인 질문이 되면 안 된다.** 반드시 선택한 목표와 입력 상황(비 오는 날)이 반영된 질문이어야 한다.

### 9. 카드 검토·승인 (보호자)

| 구분 | 내용 |
| --- | --- |
| 상단 제목 | 비 오는 날 학교 가기 |
| 상태 배지 | AI가 카드 5개를 만들었어요 |
| 카드 표시 | 이미지, 순서, 제목, 안내 문장, TTS 미리 듣기(🔊 들어보기) |
| 카드 기능 | 수정, 삭제, 순서 변경 |
| 준비물 | 카드 안 체크리스트로 표시 |
| 안내 | 확인한 카드만 아이에게 보여주세요. |
| 하단 버튼 | **아이 화면으로 시작하기** |

- 24시간 데모에서는 **카드 5개로 제한**하는 것이 좋다.
- 데모 팁: 카드 하나를 직접 수정하는 모습을 보여주면 보호자 통제가 강조된다.
  - 예: "현관에서 우산을 챙겨요." → "현관 우산꽂이에서 파란색 우산을 챙겨요."
  - 단, 개인정보 최소 수집 메시지를 유지하려면 구체적인 주소·학교명은 입력하지 않는다.

### 10. 아이 화면 전환 안내

- 별도 전체 페이지 대신 **바텀시트나 팝업**(`ParentModeSheet`)으로 처리해도 된다.
- 목적: 보호자 편집 모드 → 아동 수행 모드 전환. 승인 전 카드는 아동에게 노출되지 않음을 안내.
- 완전한 PIN 인증은 만들지 않고 **전환 UI만 구현**한다.

### 11. 아동 카드 수행

| 구분 | 내용 |
| --- | --- |
| 상단 | 진행 표시(2/5), 별 개수, 보호자 잠금(🔒) |
| 중심 요소 | **큰 행동 이미지** |
| 행동 제목 | 짧고 직접적인 문장 |
| 음성 | 자동 재생 또는 🔊 다시 듣기 |
| 체크리스트 | 필요한 카드에만 표시 |
| 주요 버튼 | **했어요!** |
| 완료 동작 | 짧은 칭찬 애니메이션 후 다음 카드 |

```
2 / 5        ⭐ 1         🔒

        [ 큰 행동 이미지 ]

여벌 양말을
가방에 넣어요

양말이 젖었을 때
갈아 신을 수 있어요.

┌──────────────────────┐
│  □ 여벌 양말           │
└──────────────────────┘

[ 🔊 다시 듣기 ]

[ 했어요! ]
```

## 미확정 사항
