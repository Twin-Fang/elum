# 이름 입력 필드 아이콘 추가 및 placeholder 정렬

## 개요
온보딩 이름 입력 화면의 텍스트 필드에 아이콘을 추가하고, placeholder 정렬을 개선했습니다. Figma 디자인 기준으로 입력 필드 좌측에 아이 호칭 아이콘을 표시하고, 사용자 입력을 더 직관적으로 안내합니다.

## 변경 사항

### 에셋 추가
- `client/assets/images/input_field_icon_child_name.svg`:
  - Figma에서 추출한 입력 필드 아이콘 SVG
  - 14×14 크기, 검정색

### 화면 구현
- `client/lib/features/onboarding/presentation/name_screen.dart`:
  - `ElumTextField`에 `leadingIconAssetPath` 파라미터 전달
  - 좌측에 아이콘 표시
  - placeholder 텍스트 중앙 정렬

### 컴포넌트 개선
- `client/lib/core/widgets/elum_text_field.dart`:
  - `leadingIconAssetPath` 파라미터 추가 (선택사항)
  - 아이콘 있을 경우 좌측 패딩 자동 조정
  - SVG 렌더링 처리

### 에셋 관리
- `client/lib/core/assets/app_assets.dart`:
  - `inputFieldIconChildName` 상수 추가

## 주요 구현 내용

### 아이콘 배치
```dart
ElumTextField(
  controller: _controller,
  hintText: '이름을 입력해주세요',
  leadingIconAssetPath: AppAssets.inputFieldIconChildName,  // 좌측 아이콘
  onChanged: ref.read(onboardingProvider.notifier).setNickname,
)
```

- 입력 필드 좌측에 14×14 아이콘 배치
- 아이콘과 텍스트 사이 8px 간격 (디자인 시스템)
- 아이콘 클릭 시에도 필드에 포커스

### Placeholder 정렬
- `textAlign: TextAlign.center` 제거 → `TextAlign.start` 유지
- 입력 후 텍스트는 자연스럽게 왼쪽 정렬
- 사용자 입력 중에도 일관된 경험 제공

### 접근성
- 아이콘은 시각적 안내용 (의미적 아이콘 아님)
- 스크린 리더에 영향 없음
- 터치 타겟 유지 (48×48 이상)

## 주의사항

### SVG 에셋 관리
- 에셋은 반드시 `AppAssets`를 통해 참조
- 문자열로 직접 경로를 쓰지 않음 (오타 방지)
- Figma에서 추가 아이콘이 나오면 같은 방식으로 추가

### 다른 입력 필드
- 아동 이름 필드만 아이콘 표시
- 다른 필드(PIN, 목표 등)는 현재 아이콘 불필요
- 나중에 필요하면 같은 로직으로 확장 가능

### 어두운 테마
- 아이콘 색상이 테마에 맞게 자동 조정되는지 확인 필수
- 현재는 밝은 테마만 구현
- 다크 모드 추가 시 아이콘 색상 토큰 분리 검토
