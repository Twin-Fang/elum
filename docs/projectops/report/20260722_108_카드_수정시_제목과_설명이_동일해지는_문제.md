# 카드 수정 시 제목과 설명이 동일해지는 문제

## 개요

카드확인 화면에서 카드를 수정해 저장하면 제목(title)과 설명(description)이 똑같아지는 문제를 수정했다. 특히 **title이 없던 예전 카드**에서 재현됐다. 백엔드 배포 로그를 확인한 결과 수정 요청(`PATCH /api/routines/{id}/steps/{stepId}`)이 서버에 도달한 흔적이 없어 **순수 프론트(클라) 문제**로 확정하고, 편집 바텀시트의 제목 초기값을 잘못 채우던 부분을 고쳤다.

## 기능 흐름

수정 전에는 제목 초기값으로 `displayTitle`(폴백 포함)을 넘겨, title 없는 카드에서 description이 제목칸으로 새어 들어갔다. 수정 후에는 실제 `title`을 넘겨, title 없는 카드는 제목칸이 비어 열린다.

```mermaid
flowchart TD
    A(["카드 수정 칩 탭"]) --> B{"카드에 title 있나?"}
    B -->|있음| C["제목칸에 title 채워 열림"]
    B -->|없음 (예전 카드)| D["제목칸 비워 열림"]
    D --> E["사용자가 제목 직접 입력"]
    C --> F{"제목·설명 모두 입력됨?"}
    E --> F
    F -->|아니오| G["저장 버튼 비활성"]
    F -->|예| H["저장 → 제목·설명 각각 반영"]
    H --> I(["제목 ≠ 설명 보장"])
```

## 변경 사항

### 버그 수정
- `client/lib/features/guardian/presentation/card_review_screen.dart`: `CardEditSheet`의 초기 제목값을 `card.displayTitle` → `card.title`로 변경. title 없는 카드는 제목칸이 비어 열려 description이 새어들지 않는다.

### 테스트
- `client/test/card_edit_sheet_test.dart`: "title 없는 예전 카드는 제목칸이 비어 열린다" 회귀 테스트 추가. title 없는 카드 수정 시 제목칸이 비어 있는지, 제목=설명으로 저장될 수 없는지 검증한다.

## 주요 구현 내용

### 근본 원인

- 서버 `RoutineStep`에는 **title 컬럼이 없다**. 따라서 `title`은 클라이언트 로컬 전용값이다.
- `ActionCard.displayTitle`은 `title.isNotEmpty ? title : description` — title이 비면 **description을 대신 반환**한다.
- 카드 수정 시트를 열 때 초기 제목으로 `card.displayTitle`을 넘겼기 때문에, title 없는 예전 카드는 **제목칸에 description이 채워진 채** 시트가 열렸다.
- 사용자가 제목을 따로 고치지 않으면 `title = description`으로 저장돼 제목·설명이 동일해졌다.

### 백엔드/프론트 판별

배포 로그를 확인한 결과 로그에는 `GET /api/routines`·`/today`·health check만 있었고 수정 요청(`PATCH .../steps/{stepId}`)이 서버에 도달한 흔적이 없었다. 서버는 정상 동작 중이었으므로 프론트 문제로 확정했다.

### 해결

편집 시트에 폴백이 섞인 `displayTitle` 대신 실제 `title`을 넘긴다. title 없는 카드는 제목칸이 비어 열리고, 저장 버튼은 제목·설명이 모두 채워져야 활성화되므로(`_canSave`) 사용자가 제목을 직접 입력하지 않으면 저장 자체가 불가능해 `title = description` 저장이 원천 차단된다.

## 주의사항

- title은 여전히 클라 로컬 전용값이라 앱 재시작·재조회 시 유실될 수 있다. title을 영구 저장하려면 서버 `RoutineStep`에 title 필드 추가가 필요하다 (관련 이슈 #71, #81).
