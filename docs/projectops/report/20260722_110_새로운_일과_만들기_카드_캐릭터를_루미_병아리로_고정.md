# 새로운 일과 만들기 카드 캐릭터를 루미 병아리로 고정

## 개요

보호자 홈 화면의 "새로운 일과 만들기" 카드 아이콘이 보호자가 온보딩에서 고른 캐릭터(고양이/여우) 배지로 잘못 렌더링되고 있었다. Figma 시안(`217:2655` 내 `217:2668` Ellipse + `217:2675` Mask group)을 다시 덤프·렌더링해 확인한 결과, 이 카드의 아이콘은 캐릭터 선택과 무관하게 민트색 원형 배경 위 고정된 AI 마스코트 "루미" 병아리 이미지여야 했다. 해당 아이콘을 Figma 기준대로 교체했다.

## 변경 사항

### 에셋
- `client/assets/images/home_new_routine_chick.svg`: Figma `217:2675` 노드에서 추출한 루미 병아리 SVG 신규 추가 (47×51)

### 에셋 상수
- `client/lib/core/assets/app_assets.dart`: `homeNewRoutineChick` 상수 추가. 기존 `characterBadgeFramed`와 값이 같아도 쓰임이 다르므로 분리했다 (client/CLAUDE.md "같은 색이어도 쓰임이 다르면 토큰을 나눈다" 규칙)

### 디자인 토큰
- `client/lib/core/theme/app_colors.dart`: `homeCardIconBg` 색상 토큰 신규 추가 (`rgba(130,220,223,0.4)`, Figma `217:2668`). 선언/생성자/`light`/`copyWith`/`lerp` 5곳 모두 반영

### 화면
- `client/lib/features/guardian/presentation/guardian_home_screen.dart`: `_NewRoutineCard`의 아이콘을 `characterBadgeFramed(character)` 정사각형 배지에서, `homeCardIconBg` 원형 배경 + `homeNewRoutineChick` SVG 조합으로 교체. 더 이상 캐릭터 정보가 필요 없어 `_NewRoutineCard`의 `character` 파라미터를 제거했다 (헤더의 캐릭터 배지는 기존 그대로 유지)

### 테스트
- `client/test/guardian_home_screen_test.dart`: "아이콘을 코드로 그리지 않고 SVG 에셋으로 렌더링한다" 테스트를 갱신 — `characterBadgeFramed`는 헤더·빈 상태 두 곳(3→2)만 기대하도록 수정하고, `homeNewRoutineChick`이 카드에 렌더링되는지 별도로 검증하는 assertion을 추가

## 주요 구현 내용

카드 아이콘은 원형 배경(56×56)과 병아리 SVG(47×51)를 겹쳐서 구성했다. 배경은 단색+투명도 조합이라 client/CLAUDE.md 규칙에 따라 코드(`DecoratedBox` + `BoxShape.circle`)로 그렸고, 병아리 자체는 도형이 복잡해 SVG 에셋으로 받았다.
