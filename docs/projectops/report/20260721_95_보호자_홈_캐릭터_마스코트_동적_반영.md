# 🎨[디자인][클라][홈] 보호자 홈 캐릭터 마스코트를 온보딩 선택값으로 동적 반영

## 개요

보호자 홈 화면(Figma `보호자_홈` 217:2655)의 마스코트 아이콘 3곳(헤더 우측 상단 캐릭터 배지, "새로운 일과 만들기" 카드 일러스트, "오늘 일과" 빈 상태 일러스트)이 온보딩에서 보호자가 고른 캐릭터와 무관하게 고정된 정적 SVG를 쓰고 있었다. 아이 홈 화면에 이미 구현되어 있던 캐릭터별 배지 에셋 함수를 재사용해, 이 3곳 모두 온보딩 선택값(고양이 `루루` / 여우 `포포`)을 그대로 반영하도록 바꿨다.

## 변경 사항

### 에셋 정리
- `client/lib/core/assets/app_assets.dart`: 더는 쓰이지 않는 홈 전용 정적 에셋 상수 3개(`homeCharacterBadge`, `homeNewRoutineIllust`, `homeEmptyIllust`) 제거. 캐릭터별 테두리 배지를 반환하는 `characterBadgeFramed(CardCharacter)`(고양이는 파랑 `#9CADF1`, 여우는 주황 `#EB9B73`)를 그대로 재사용한다.

### 보호자 홈 화면
- `client/lib/features/guardian/presentation/guardian_home_screen.dart`: 헤더 우측 상단 캐릭터 배지와 "새로운 일과 만들기" 카드 일러스트에 `AppAssets.characterBadgeFramed(character)`를 적용. `character`는 `ref.watch(onboardingProvider).cardCharacter ?? CardCharacter.cat`으로, 캐릭터를 아직 선택하지 않은 레거시 데이터는 아이 홈과 동일하게 고양이로 폴백한다.
- `client/lib/features/guardian/presentation/widgets/today_routine_section.dart`: "오늘 일과" 빈 상태(`EmptyRoutines`)의 40×40 일러스트도 동일한 방식으로 `characterBadgeFramed(character)`로 교체.

### 테스트
- `client/test/guardian_home_screen_test.dart`: 정적 에셋 상수를 참조하던 기대값을 `characterBadgeFramed(CardCharacter.cat)` 기준으로 갱신. 일과 0건 상태에서는 헤더·새 일과 카드·빈 상태 3곳에 같은 캐릭터 배지가 렌더링되므로 `findsNWidgets(3)`으로 고정.
- `client/test/mode_switch_entry_test.dart`: 헤더 배지가 "새로운 일과 만들기" 카드와 같은 SVG 에셋을 공유하게 되면서 `AppPressable` 조상 탐색이 2곳(둘 다 눌러지는 위젯)에서 매칭되는 문제를 발견. 화면 최상단(헤더)이 먼저 렌더링된다는 점을 이용해 `.first`로 특정해 해결.

## 주요 구현 내용

새 에셋을 만들지 않고, 아이 홈 화면(`child_home_screen.dart`)에 이미 구현되어 있던 `AppAssets.characterBadgeFramed(CardCharacter)`를 보호자 홈 3개 지점에 그대로 재사용했다. 온보딩 상태(`onboardingProvider`)를 두 화면에서 동일한 방식(`cardCharacter ?? CardCharacter.cat`)으로 읽기 때문에, 보호자가 온보딩에서 고른 캐릭터가 앱 전역에서 일관되게 보인다.

## 주의사항

- 일과가 0건일 때는 같은 캐릭터 배지 에셋이 화면에 3번(헤더·새 일과 카드·빈 상태) 나타난다. 위젯 테스트에서 `AppPressable` 조상만으로 특정 배지를 구분해야 하는 경우 `.first` 등으로 명시적으로 좁혀야 한다 — `mode_switch_entry_test.dart`에서 실제로 이 문제를 겪었다.
- `flutter test` 전체 스위트에서 `child_mode_test.dart`(별 렌더링, PIN 검증), `design_token_test.dart` 등 4~5건이 실패하는데, 이번 변경 이전부터 실패하던 것으로(다른 세션이 작업 중인 `card_review_screen.dart` 관련) 이 작업과는 무관함을 `git stash`로 확인했다.
