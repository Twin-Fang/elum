# 온보딩_목표 화면 Figma 정합 및 선택 색 토큰 분리

## 개요
온보딩 목표 선택 화면을 Figma 디자인과 정확히 맞추기 위해 버튼 색상을 세분화하고, 선택·비선택·활성 상태를 명확히 구분했습니다. 색상 토큰을 분리하여 각 상태의 의도를 명확히 하고, 나중에 한쪽만 수정할 수 있도록 설계했습니다.

## 변경 사항

### 색상 토큰 분리
- `client/lib/core/theme/app_colors.dart`:
  - `goalNotSelected`: 선택 안 된 목표 버튼 배경색
  - `goalSelected`: 선택된 목표 버튼 배경색 (주황색)
  - `goalDisabled`: 비활성화된 목표 버튼 배경색
  - 각 상태의 텍스트 색상도 함께 정의

### 화면 구현
- `client/lib/features/onboarding/presentation/goals_screen.dart`:
  - 각 목표 버튼에 상태별 색상 토큰 적용
  - 선택 여부에 따라 배경색·테두리·텍스트 색 동적 변경
  - Figma의 버튼 스타일(`enabled`, `selected`, `disabled`) 정확히 재현

### 문서
- `client/docs/design-system.md`:
  - 목표 버튼 색상 명세 추가
  - HEX 값, 사용 상황 명기
  - 토큰 분리 원칙 설명 추가

## 주요 구현 내용

### 색상 토큰 설계 원리
```dart
// ❌ 같은 색이라고 합치지 않는다
goalNotSelected: Color(0xFFF5F5F5),
homeCardBackground: Color(0xFFF5F5F5),  // 같은 값이어도 따로

// ✅ 이 둘은 항상 같이 바뀌는가? 아니면 따로다
// 목표 버튼만 바뀌어야 할 상황이 있을 수 있으므로 분리 유지
```

### 상태별 렌더링
- **비선택**: 밝은 배경, 검정 텍스트
- **선택**: 주황 배경, 흰 텍스트
- **비활성**: 회색 배경, 회색 텍스트

각 상태는 `ref.watch(onboardingProvider)`로 선택 여부 감지하여 즉시 반영됩니다.

### Figma 정합 검증
원본 Figma 노드(`204:1069` 온보딩_목표)의 버튼 4개(보조·준비·새로·독립)를 각각 확인:
- 크기: 160×60 (고정)
- 색상: 선택 시 주황(`#FF9D4D`), 미선택 시 연회색(`#F5F5F5`)
- 타이포그래피: 16sp, w400

## 주의사항

### 색상 토큰 유지
- 새로운 버튼 상태가 생기면 **HEX를 추측하지 말고** Figma 컴포넌트셋에서 확인
- 같은 색이어도 쓰임이 다르면 토큰을 나눈다 (§ 토큰 분리 원칙)

### 테스트 검증
- 위젯 테스트: 선택 전·후 색상 변경 확인
- 골든 이미지: Figma와 렌더링 결과 비교 (발표 직전 최종 확인)

### 아동 화면과의 일관성
- 이 색상 토큰은 보호자 모드만 사용
- 아동 모드 버튼 색상은 별도 토큰으로 분리 유지
