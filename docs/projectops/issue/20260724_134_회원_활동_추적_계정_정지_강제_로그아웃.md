📝 현재 문제점
---

- 회원이 언제 마지막으로 로그인했는지, 서비스를 실제로 쓰고 있는지 알 수 없습니다.
- 문제가 되는 계정을 차단할 방법이 없어 구독제 전환이나 어뷰징 대응이 불가능합니다.
- JWT가 stateless라 발급된 토큰을 무효화할 수단이 없습니다.

🛠️ 해결 방안 / 제안 기능
---

- 회원에 상태(활성, 정지), 마지막 로그인 시각, 마지막 활동 시각, 로그인 횟수를 기록합니다.
- 로그인 시 정지 계정은 차단하고, 활동 시각은 인증 요청마다 60초 스로틀로 갱신합니다.
- 관리자가 회원을 정지, 해제할 수 있고, 정지된 회원은 로그인과 API 사용이 모두 차단됩니다.
- 강제 로그아웃: 특정 시각 이전에 발급된 토큰을 거부하는 기준 시각을 도입해, 관리자가 누르면 해당 회원의 기존 토큰이 전부 무효화됩니다.
- 회원 상태 확인 중 DB 오류가 나도 일반 요청은 죽지 않게 가용성을 우선합니다.

⚙️ 작업 내용
---

- 회원 테이블에 상태, 활동 컬럼 추가(Flyway 마이그레이션)
- 로그인 흐름에 정지 차단과 활동 기록 연동
- JWT 인증 필터에 회원 상태, 토큰 발급 시각 검증 및 활동 갱신(스로틀) 추가
- 관리자 회원 정지, 해제, 강제 로그아웃 액션 추가
- 접근 가드 단위 테스트
