📝 현재 문제점
---

- Gemini 텍스트 모델명, 이미지 모델명이 환경변수(yml)에 고정되어 있어 모델을 바꾸려면 재배포가 필요합니다.
- 이미지 생성 비율(aspectRatio), temperature 등 생성 파라미터가 코드에 하드코딩되어 있어 운영 중 조정이 불가능합니다.
- 내부 DLP용 로컬 LLM(Ollama) 모델명도 동일하게 고정되어 있습니다.
- 프롬프트는 이미 관리자 화면에서 동적으로 관리되지만, 모델과 파라미터는 그렇지 못해 관리 체계가 반쪽짜리입니다.

🛠️ 해결 방안 / 제안 기능
---

- `system_config` 테이블을 단일 진실 공급원으로 하는 동적 설정 관리 체계를 도입합니다.
- 서버는 설정을 메모리 캐시(TTL 30초)에 올려 쓰고, 값 변경 시 즉시 반영합니다. 다중 레플리카도 TTL로 수렴합니다.
- 관리자 페이지에 "시스템 설정" 화면을 추가해 그룹별(Gemini 텍스트, Gemini 이미지, 로컬 LLM, 요금)로 조회, 수정, 기본값 복원이 가능하게 합니다.
- 관리 대상: Gemini 텍스트 모델명, temperature, Gemini 이미지 모델명, 이미지 비율(선택지 제공), 로컬 LLM 모델명, AI 요금 단가(토큰당 비용 계산용).

⚙️ 작업 내용
---

- system_config 테이블 신설(Flyway 마이그레이션) 및 기본값 시딩 로직
- 타입 검증(정수, 소수, 선택지) 포함 설정 서비스와 TTL 메모리 캐시
- Gemini 텍스트, 이미지 클라이언트와 DLP 서비스가 설정값을 읽도록 연동
- 관리자 시스템 설정 화면(조회, 수정, 기본값 복원, 변경됨 표시)
- 설정 서비스 단위 테스트
